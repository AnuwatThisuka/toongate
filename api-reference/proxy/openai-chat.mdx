---
title: "POST /v1/chat/completions"
api: "POST https://toongate.workers.dev/v1/chat/completions"
description: OpenAI-compatible chat completions with automatic TOON compression.
---

## Overview

This endpoint proxies to the OpenAI Chat Completions API (or any OpenAI-compatible upstream set via `UPSTREAM_URL`). toongate compresses eligible structured content in `messages[].content` before forwarding the request.

The response is identical to what OpenAI returns — including streaming SSE when `stream: true`.

## Authorization

<ParamField header="Authorization" type="string">
  Required only when `PROXY_AUTH_KEY` is set on the Worker. Pass `Bearer <PROXY_AUTH_KEY>` to authenticate to toongate itself. The upstream OpenAI key (`OPENAI_API_KEY`) is always injected automatically — you do not need to pass it.
</ParamField>

## Request body

The request body is the standard [OpenAI Chat Completions request](https://platform.openai.com/docs/api-reference/chat/create). All fields pass through unchanged.

<ParamField body="model" type="string" required>
  Model ID (e.g. `gpt-4o`, `gpt-4o-mini`).
</ParamField>

<ParamField body="messages" type="array" required>
  Array of message objects. `content` must be an array (not a plain string) for structured data to be eligible — use OpenAI multimodal format with `{ type: "text", text: "..." }` parts.
</ParamField>

<ParamField body="stream" type="boolean">
  When `true`, returns a stream of SSE events. toongate compresses the request payload and passes chunks back as-is.
</ParamField>

## Response headers

| Header | Description |
|---|---|
| `X-Toongate-Compressed` | `true` if the request payload was compressed |
| `X-Toongate-Tokens-Before` | Estimated token count before compression |
| `X-Toongate-Tokens-After` | Estimated token count after compression |
| `X-Toongate-Tokens-Saved` | Tokens saved |
| `X-Toongate-Eligibility-Score` | Eligibility score (0–1) |

<RequestExample>
```bash cURL
curl https://toongate.workers.dev/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $PROXY_AUTH_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {
        "role": "user",
        "content": [
          { "type": "text", "text": "Summarize:" },
          { "type": "text", "text": "[{\"id\":1,\"title\":\"A\",\"score\":0.9},{\"id\":2,\"title\":\"B\",\"score\":0.8}]" }
        ]
      }
    ]
  }'
```
</RequestExample>

```json 200
    }
  ],

}
```
</ResponseExample>
