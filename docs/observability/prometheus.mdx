## Native /metrics endpoint

toongate exposes a Prometheus-compatible `/metrics` endpoint protected by `ADMIN_KEY`:

```bash
curl https://toongate.workers.dev/metrics \
  -H "X-Toongate-Admin-Key: <ADMIN_KEY>"
```

Example response:

```
# HELP toongate_requests_total Total proxy requests processed
# TYPE toongate_requests_total counter
toongate_requests_total 4821

# HELP toongate_tokens_saved_total Total input tokens saved by compression
# TYPE toongate_tokens_saved_total counter
toongate_tokens_saved_total 310420

# HELP toongate_usd_saved_total Estimated USD saved by compression
# TYPE toongate_usd_saved_total gauge
toongate_usd_saved_total 4.821

# HELP toongate_compressed_requests_total Requests where TOON compression was applied
# TYPE toongate_compressed_requests_total counter
toongate_compressed_requests_total 3105

# HELP circuit_breaker_tripped Whether the circuit breaker is currently open (1=open, 0=closed)
# TYPE circuit_breaker_tripped gauge
circuit_breaker_tripped 0

# HELP circuit_breaker_errors_in_window Number of errors in the current circuit breaker window (last 20 requests)
# TYPE circuit_breaker_errors_in_window gauge
circuit_breaker_errors_in_window 0
```

---

## Available metrics

| Metric | Type | Description |
|---|---|---|
| `toongate_requests_total` | counter | Total proxy requests processed (all routes) |
| `toongate_tokens_saved_total` | counter | Cumulative input tokens saved by TOON compression |
| `toongate_usd_saved_total` | gauge | Estimated USD saved (recalculated from savings log) |
| `toongate_compressed_requests_total` | counter | Requests where compression was actually applied |
| `circuit_breaker_tripped` | gauge | `1` if circuit breaker is currently open, `0` if closed |
| `circuit_breaker_errors_in_window` | gauge | Error count in the current 20-request sliding window |

Aggregate metrics (`requests_total`, `tokens_saved_total`, `usd_saved_total`) are computed from the D1 savings log. Circuit breaker metrics reflect the live in-memory state of the running isolate.

## Prometheus scrape config

Because Prometheus's native `authorization` block sends `Authorization: Bearer` but `/metrics` requires `X-Toongate-Admin-Key`, use a lightweight scrape adapter Worker:

```typescript
// scrape-adapter.ts — deploy as a separate Cloudflare Worker
export default {
  async fetch(req: Request, env: { ADMIN_KEY: string; TOONGATE_URL: string }) {
    const resp = await fetch(`${env.TOONGATE_URL}/metrics`, {
      headers: { "X-Toongate-Admin-Key": env.ADMIN_KEY },
    });
    return new Response(resp.body, {
      headers: { "Content-Type": "text/plain; version=0.0.4" },
    });
  },
};
```

Then point Prometheus at the adapter:

```yaml
scrape_configs:
  - job_name: toongate
    scrape_interval: 60s
    static_configs:
      - targets: ["scrape-adapter.yourworker.workers.dev"]
    scheme: https
```

## Grafana

1. Add a Prometheus data source pointing at your Prometheus instance (or scrape adapter).
2. Use these PromQL queries:

```promql
# Compression rate
toongate_compressed_requests_total / toongate_requests_total

# Token savings rate over time
rate(toongate_tokens_saved_total[5m])

# Circuit breaker health
circuit_breaker_tripped
```

## Grafana Cloud with Infinity data source

If you prefer to query toongate directly without Prometheus:

1. Install the [Infinity plugin](https://grafana.com/grafana/plugins/yesoreyeram-infinity-datasource/)
2. Add a new Infinity data source
3. Create a JSON query pointing at `https://toongate.workers.dev/savings/summary` with the `X-Toongate-Admin-Key` header

For live circuit breaker state, add a second panel querying `/metrics` with text/plain parsing.
