## What is toongate?

toongate sits in front of any LLM gateway and compresses structured data in your prompts before it reaches your provider — then passes the response back unchanged.

Your app never knows it happened.

```diff
- const openai = new OpenAI({
-   baseURL: "https://gateway.ai.cloudflare.com/v1/{account}/{gateway}/openai",
- });
+ const openai = new OpenAI({
+   baseURL: "https://toongate.workers.dev/v1",
+ });
```

One `baseURL` change. No new headers. No SDK changes. Every gateway feature you already rely on keeps working.

## How it saves tokens

toongate uses the [TOON format](https://toonformat.dev) to encode uniform arrays of objects — the kind that repeat field names for every row.

<CardGroup cols={2}>
  <Card title="Before (JSON)" icon="file-code">
    ```json
    [
      {"id": 1, "name": "Alice", "score": 0.91},
      {"id": 2, "name": "Bob",   "score": 0.87},
      {"id": 3, "name": "Carol", "score": 0.79}
    ]
    ```
    **~38 tokens**
  </Card>
  <Card title="After (TOON)" icon="bolt">
    ```
    {id,name,score}[3]
    1|Alice|0.91
    2|Bob|0.87
    3|Carol|0.79
    ```
    **~22 tokens — 42% fewer**
  </Card>
</CardGroup>

Free-form text, deeply nested configs, and anything that doesn't compress well passes through unchanged. toongate only encodes when it helps.

## Compression at a glance

| Payload type | Savings | Example |
|---|---|---|
| Uniform array of objects | **~40–48%** | RAG chunks, DB rows, product catalogs |
| Mixed structured data | ~20–30% | Prompts with both tables and nested objects |
| Free-form text | 0% (pass-through) | Plain Q&A, summaries, creative prompts |
| Deeply nested JSON | 0% (pass-through) | Complex config objects |

Accuracy is slightly _higher_ with TOON — explicit length markers give models a clearer schema. See [benchmarks](https://toonformat.dev/guide/benchmarks.html).

## Benchmarks

> Tested on real traffic · Cloudflare Workers (Singapore edge) ·
> 50 requests per payload · concurrency 10

| Payload | Tokens saved | Saving % | Compressed |
|---|---|---|---|
| DB rows (10 rows) | 177 tokens/req | **48.6%** | 100% |
| Product catalog (8 rows) | 108 tokens/req | **41.4%** | 100% |
| RAG chunks (5 rows) | 32 tokens/req | 12.0%† | 100% |
| Plain text | 0 tokens/req | 0% | 0% (pass-through ✓) |

† RAG savings scale with row count — 10+ rows yields ~40%.
The sweet spot is uniform arrays with 10 or more objects.

**From 100 mixed requests:**
- 3,672 tokens saved total
- $0.0092 estimated cost reduction (gpt-4o-mini pricing)
- 74.2% of requests compressed
