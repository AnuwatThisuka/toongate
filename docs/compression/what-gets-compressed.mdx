## The sweet spot: uniform arrays

TOON excels at encoding arrays where every object has the same keys — because standard JSON repeats field names for every single row.

The more rows, the more savings. A 100-row array saves proportionally more than a 3-row one.

### Moderate compression (20–30%)

Mixed content — prompts that combine structured tables with free-form text sections. The array portion compresses; the text passes through.

### No compression (pass-through)

| Payload type | Why |
|---|---|
| Plain Q&A | No array structure to encode |
| Creative prompts | Free-form text only |
| Single objects (`{}`) | No repeated keys across rows |
| Deeply nested configs | Non-uniform structure |
| Already-compressed content | Would increase size |

## Tips to maximize savings

**1. Use a content array — not a plain string**

The eligibility scorer inspects `messages[].content` as structured data. Plain strings always score `0.0` and pass through unchanged — even if they contain JSON text.

```typescript
// ✅ Good — content is an array (OpenAI multimodal format)
{
  role: "user",
  content: [
    { type: "text", text: "Summarize these search results:" },
    { type: "text", text: JSON.stringify(myDataArray) },
  ],
}

// ❌ Won't compress — plain string scores 0.0
{ role: "user", content: JSON.stringify(myDataArray) }
{ role: "user", content: `Here is the data: ${JSON.stringify(myDataArray)} please analyze` }
```

**2. Keep object keys consistent across rows**

TOON requires uniform keys. If some rows have extra fields, split into two arrays.

```json
// ✅ Compresses well — uniform keys
[
  { "id": 1, "name": "Alice", "score": 0.9 },
  { "id": 2, "name": "Bob",   "score": 0.8 }
]

// ⚠️ Lower compression — inconsistent keys
[
  { "id": 1, "name": "Alice", "score": 0.9, "tags": ["vip"] },
  { "id": 2, "name": "Bob",   "score": 0.8 }
]
```

**3. More rows = better ratio**

The TOON header (`{field1,field2,...}[N]`) is a fixed overhead. The more rows you have, the smaller it is relative to the savings.

| Rows | Typical token savings |
|---|---|
| 3–5 | 20–30% |
| 10–20 | 35–40% |
| 50+ | 40–45% |

## Checking what compressed

After any request, check the response headers:

```bash
curl -si https://toongate.workers.dev/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $PROXY_AUTH_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [{
      "role": "user",
      "content": [
        { "type": "text", "text": "Summarize:" },
        { "type": "text", "text": "[{\"id\":1,\"title\":\"A\",\"score\":0.9}]" }
      ]
    }]
  }' | grep -i x-toongate

# x-toongate-compressed: true
# x-toongate-tokens-before: 1420
# x-toongate-tokens-after: 851
# x-toongate-tokens-saved: 569
# x-toongate-eligibility-score: 0.84
```

If `X-Toongate-Compressed: false` with a low eligibility score, your payload is free-form text and there's nothing to compress — that's expected and correct.
