## Overview

toongate's `/bedrock/v1/messages` endpoint accepts standard Anthropic-format requests and forwards them to AWS Bedrock Anthropic Claude models. SigV4 signing happens automatically inside the Worker — your app never handles AWS credentials.

---

## Setup

### 1. Configure AWS credentials

```bash
wrangler secret put AWS_ACCESS_KEY_ID
wrangler secret put AWS_SECRET_ACCESS_KEY
# Optional — for temporary STS credentials
wrangler secret put AWS_SESSION_TOKEN
```

```bash
# In wrangler.jsonc (non-sensitive)
AWS_REGION = "us-east-1"   # default when unset
```

The IAM principal needs the following permissions:

```json
{
  "Effect": "Allow",
  "Action": "bedrock:InvokeModel",
  "Resource": "arn:aws:bedrock:*::foundation-model/anthropic.*"
}
```

### 2. Point your SDK at toongate

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({
  baseURL: "https://toongate.workers.dev/bedrock/v1",
  apiKey: "<PROXY_AUTH_KEY>",  // toongate proxy auth, not Anthropic key
});

const msg = await client.messages.create({
  model: "anthropic.claude-3-5-sonnet-20241022-v2:0",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello!" }],
});
```

---

## Supported route

| Method | Path | Description |
|---|---|---|
| `POST` | `/bedrock/v1/messages` | Anthropic Messages API → Bedrock |

The `model` field must be a valid Bedrock Anthropic model ID, e.g.:
- `anthropic.claude-3-5-sonnet-20241022-v2:0`
- `anthropic.claude-3-haiku-20240307-v1:0`
- `anthropic.claude-3-opus-20240229-v1:0`

---

## How it works

1. toongate receives the standard Anthropic request body.
2. TOON compression is applied to the `messages` array (subject to your threshold).
3. `anthropic_version: "bedrock-2023-05-31"` is injected and `model` is removed from the body.
4. The request is SigV4-signed using `crypto.subtle` (Web Crypto — no Node.js SDK needed).
5. The signed request is forwarded to `https://bedrock-runtime.{region}.amazonaws.com/model/{modelId}/invoke`.
6. Any TOON-encoded content in the response is decoded transparently.

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `AWS_ACCESS_KEY_ID` | — | Required. AWS access key. |
| `AWS_SECRET_ACCESS_KEY` | — | Required. AWS secret key. |
| `AWS_REGION` | `us-east-1` | AWS region where Bedrock is enabled. |
| `AWS_SESSION_TOKEN` | — | Optional. For STS temporary credentials. |

All AWS credentials should be set via `wrangler secret put` — never commit them to `wrangler.jsonc`.

---

## Using temporary credentials (STS)

If you rotate credentials via AWS STS or IAM roles, pass `AWS_SESSION_TOKEN` alongside the key pair. toongate includes it as the `x-amz-security-token` header automatically when set.

For fully automated rotation, redeploy the Worker or use a secrets manager that updates Cloudflare Worker secrets via the Cloudflare API.

---

## Notes

- Streaming (`stream: true`) is not currently supported on the Bedrock route — Bedrock's streaming API uses a different response format. Non-streaming responses work for all token lengths.
- The Bedrock endpoint is protected by `PROXY_AUTH_KEY` — all requests require `Authorization: Bearer <PROXY_AUTH_KEY>`.
- SigV4 signing adds ~1 ms of CPU overhead per request (Web Crypto is hardware-accelerated on Cloudflare Workers).
