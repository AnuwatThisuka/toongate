## How it works

toongate exposes a dedicated prefix `/azure/v1/*` for Azure OpenAI. It builds the deployment-specific Azure URL from the `model` field in the request body, injects your `AZURE_OPENAI_API_KEY` as the `api-key` header, and forwards the (optionally compressed) payload to Azure.

```
Your App  →  toongate /azure/v1/chat/completions  →  {resource}.openai.azure.com/openai/deployments/{model}/chat/completions?api-version=…
```

No SDK changes beyond `baseURL` — Azure-specific URL building and auth happen inside toongate.

---

## Setup

Set the following secrets on your Worker:

```bash
wrangler secret put AZURE_OPENAI_API_KEY      # Azure API key
wrangler secret put AZURE_OPENAI_ENDPOINT     # e.g. https://my-resource.openai.azure.com
wrangler secret put AZURE_OPENAI_API_VERSION  # optional — default: 2024-02-01
```

For local development, add them to `.dev.vars`:

```ini
AZURE_OPENAI_API_KEY=...
AZURE_OPENAI_ENDPOINT=https://my-resource.openai.azure.com
AZURE_OPENAI_API_VERSION=2024-02-01
```

## Usage

Point your SDK at `https://toongate.workers.dev/azure/v1`. Pass your **Azure deployment name** as the `model` field — toongate uses it to build the upstream URL.

```typescript
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://toongate.workers.dev/azure/v1",
  apiKey: "unused", // toongate injects AZURE_OPENAI_API_KEY automatically
});

const response = await client.chat.completions.create({
  model: "my-gpt4o-deployment", // ← your Azure deployment name
  messages: [
    {
      role: "user",
      content: [
        { type: "text", text: "Summarize:" },
        { type: "text", text: JSON.stringify(myDataArray) },
      ],
    },
  ],
});
```

## Supported routes

| Method | toongate path | Azure upstream |
|--------|--------------|----------------|
| `POST` | `/azure/v1/chat/completions` | `{endpoint}/openai/deployments/{model}/chat/completions?api-version={ver}` |
| `POST` | `/azure/v1/embeddings` | `{endpoint}/openai/deployments/{model}/embeddings?api-version={ver}` |

---

## Authentication

Azure OpenAI uses the `api-key` header (not `Authorization: Bearer`). toongate handles this automatically — you do not need to set any auth headers in your SDK.

```
Your App (no key) → toongate (injects api-key header) → Azure OpenAI
```

---

## Streaming

Streaming (`stream: true`) is fully supported. toongate compresses the request payload and passes SSE chunks back to your client as they arrive.

```typescript
const stream = await client.chat.completions.create({
  model: "my-gpt4o-deployment",
  messages: [{ role: "user", content: "..." }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content ?? "");
}
```

## Per-route threshold

Azure routes share the same threshold knobs as OpenAI:

```bash
TOON_THRESHOLD_CHAT=0.5        # applies to /azure/v1/chat/completions
TOON_THRESHOLD_EMBEDDINGS=0.8  # applies to /azure/v1/embeddings
```

## Error responses

| Status | Condition |
|--------|-----------|
| `401` | `PROXY_AUTH_KEY` is set but the request is missing or has a wrong `Authorization: Bearer` header |
| `503` | `AZURE_OPENAI_ENDPOINT` or `AZURE_OPENAI_API_KEY` is not configured |
| `504` | Upstream did not respond within `UPSTREAM_TIMEOUT_MS` (default 30 s) |
| `502` | Any other upstream fetch error |
